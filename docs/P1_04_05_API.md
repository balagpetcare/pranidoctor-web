## 2. Frozen envelope (all six routes)

```json
{ "ok": true, "data": { } }
{ "ok": false, "error": { "code": "STRING", "message": "STRING", "details": {} } }
```

Transport: **httpOnly session cookies** (not Bearer).

| Panel | Cookie name |
|-------|-------------|
| Doctor | `prani_doctor_session` |
| Technician | `prani_technician_session` |

## 3. Doctor panel

### 3.1 `POST /api/doctor/auth/login`

**Request (frozen):**

```json
{
  "email": "doctor@example.com",
  "password": "secret"
}
```

**Success 200:**

```json
{
  "ok": true,
  "data": {
    "user": {
      "id": "cuid",
      "email": "doctor@example.com",
      "displayName": "Dr. Example",
      "name": "Dr. Example",
      "role": "DOCTOR"
    }
  }
}
```

**Set-Cookie:** `prani_doctor_session=<JWT>` (httpOnly, SameSite=lax, path=/).

**Errors (frozen codes):**

| Code | HTTP | When |
|------|------|------|
| `INVALID_JSON` | 400 | Bad body |
| `VALIDATION_ERROR` | 422 | Zod fail (email/password) |
| `INVALID_CREDENTIALS` | 401 | Wrong email/password or not eligible doctor |
| `SERVER_MISCONFIGURED` | 500 | JWT secret / token issue |
| `db_unavailable` | 503 | DB connectivity (P1-06 pattern) |
| `server_error` | 500 | Unexpected Prisma error |

**Login gate (unchanged):**

- `User.role === DOCTOR`
- `User.status === ACTIVE`
- `DoctorProfile` exists
- `DoctorProfile.providerStatus === ACTIVE`

**P1-04 side effect:** `UserSession` row created (`channel=DOCTOR_PANEL`) — no response change.

---

### 3.2 `POST /api/doctor/auth/logout`

**Request:** empty body; cookie sent.

**Success 200:**

```json
{
  "ok": true,
  "data": {
    "signedOut": true
  }
}
```

**Set-Cookie:** clears `prani_doctor_session`.

**P1-04 addition (behavior only):** revoke latest ACTIVE `UserSession` for doctor panel + audit `LOGOUT` (already present).

---

### 3.3 `GET /api/doctor/auth/me`

**Auth:** `prani_doctor_session` cookie.

**Success 200 — current (P1-03):**

```json
{
  "ok": true,
  "data": {
    "user": {
      "userId": "cuid",
      "doctorProfileId": "cuid",
      "email": "doctor@example.com",
      "displayName": "Dr. Example"
    }
  }
}
```

**Success 200 — target (P1-04 + P1-05):**

```json
{
  "ok": true,
  "data": {
    "user": {
      "id": "cuid",
      "doctorProfileId": "cuid",
      "email": "doctor@example.com",
      "displayName": "Dr. Example",
      "role": "DOCTOR",
      "providerStatus": "ACTIVE"
    }
  }
}
```

| Change | Type | Breaking? |
|--------|------|-----------|
| `userId` → `id` | **Normalization** | No — web already expects `id`; fixes compat bug |
| `role` | Additive optional | No |
| `providerStatus` | Additive optional | No |

**Errors (frozen):**

| Code | HTTP |
|------|------|
| `UNAUTHORIZED` | 401 — no/invalid cookie |
| `FORBIDDEN` | 403 — not active doctor / profile |

**Web consumer:** `pranidoctor-web/src/lib/doctor-auth/panel-access.ts` — requires `data.user.id`.

## 4. Technician panel

### 4.1 `POST /api/technician/auth/login`

**Request:**

```json
{
  "email": "tech@example.com",
  "password": "secret"
}
```

**Success 200:**

```json
{
  "ok": true,
  "data": {
    "user": {
      "id": "cuid",
      "email": "tech@example.com",
      "displayName": "Tech Example",
      "name": "Tech Example",
      "role": "AI_TECHNICIAN"
    }
  }
}
```

**Set-Cookie:** `prani_technician_session=<JWT>`.

**Errors:** same family as doctor (`INVALID_CREDENTIALS`, etc.).

**Login gate:**

- `User.role === AI_TECHNICIAN`
- `User.status === ACTIVE`
- `AiTechnicianProfile` exists
- `AiTechnicianProfile.providerStatus === ACTIVE`

---

### 4.2 `POST /api/technician/auth/logout`

Same pattern as doctor — `{ signedOut: true }`, cookie cleared, session revoke (P1-04).

---

### 4.3 `GET /api/technician/auth/me`

## 7. Permissions & audit

### 7.1 Permissions

Doctor/technician routes **do not** call `assertAdminCan` / `permissions.registry.ts`.

Enforcement:

- **Auth routes:** panel service login gates (role + profile + provider status).
- **Business routes:** `requireDoctorApiActor` / `requireTechnicianApiActor` in `legacy/web/lib/*-auth/api-guard.ts`.

### 7.2 Audit events

| Action | Doctor channel | Technician channel |
|--------|----------------|-------------------|
| `LOGIN_SUCCESS` | `doctor_panel` | `technician_panel` |
| `LOGIN_FAILURE` | `doctor_panel` | `technician_panel` |
| `LOGOUT` | `doctor_panel` | `technician_panel` |
| `SESSION_REVOKED` | on logout revoke (P1-04) | on logout revoke (P1-04) |
