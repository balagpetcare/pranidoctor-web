## 4. Models

### 4.1 `UserDevice`

```prisma
model UserDevice {
  id           String    @id @default(cuid())
  userId       String
  user         User      @relation(fields: [userId], references: [id], onDelete: Cascade)

  deviceKey    String    // client-stable identifier
  platform     String?   // android | ios | web
  pushToken    String?
  appVersion   String?
  lastActiveAt DateTime  @default(now())
  revokedAt    DateTime?
  createdAt    DateTime  @default(now())
  updatedAt    DateTime  @updatedAt

  sessions     UserSession[]

  @@unique([userId, deviceKey])
  @@index([userId])
  @@map("user_devices")
}
```

| Rule | Detail |
|------|--------|
| Upsert key | `(userId, deviceKey)` |
| Revoke | Set `revokedAt`; filter `revokedAt IS NULL` for active |
| Push | `pushToken` updated on register |

### 4.2 `UserSession`

Aligned with archived foundation patterns (`user_sessions`) but using current Prisma `cuid` convention:

```prisma
model UserSession {
  id            String        @id @default(cuid())
  userId        String
  user          User          @relation(fields: [userId], references: [id], onDelete: Cascade)

  channel       AuthChannel
  status        SessionStatus @default(ACTIVE)

  deviceId      String?
  device        UserDevice?   @relation(fields: [deviceId], references: [id], onDelete: SetNull)

  ipAddress     String?
  userAgent     String?

  expiresAt     DateTime
  lastSeenAt    DateTime      @default(now())
  revokedAt     DateTime?
  revokedReason String?

  createdAt     DateTime      @default(now())
  updatedAt     DateTime      @updatedAt

  refreshTokens RefreshToken[]

  @@index([userId, channel])
  @@index([userId, status])
  @@index([expiresAt])
  @@map("user_sessions")
}
```

| Rule | Detail |
|------|--------|
| Create | On mobile OTP verify, password login, panel login (P1-06 row only) |
| Revoke single | `status=REVOKED`, `revokedAt`, `revokedReason` |
| Revoke all | `updateMany` where `userId` and `status=ACTIVE` |
| Expire job | Phase 2 cron — Phase 1 lazy check on refresh (`expiresAt < now()` → `EXPIRED`) |

**Note:** Archived foundation used `deletedAt` soft-delete on sessions. Current product schema uses `AuthAuditEvent` for trail; **P1-06 uses `revokedAt` only** (no soft-delete on session) to match PHASE1_DB_MAP simplicity. If ops require soft-delete later, add `deletedAt` additively.

### 4.3 `RefreshToken`

Matches rotation design from `_archived_foundation` (proven in migration-report):

```prisma
model RefreshToken {
  id           String    @id @default(cuid())
  userId       String
  user         User      @relation(fields: [userId], references: [id], onDelete: Cascade)

  sessionId    String
  session      UserSession @relation(fields: [sessionId], references: [id], onDelete: Cascade)

  tokenHash    String    @unique
  context      AuthChannel @default(MOBILE)  // or String "mobile"

  deviceId     String?
  device       UserDevice? @relation(fields: [deviceId], references: [id], onDelete: SetNull)

  expiresAt    DateTime
  revoked      Boolean   @default(false)
  revokedAt    DateTime?
  rotatedAt    DateTime?
  rotatedToId  String?   // self-FK optional — link to replacement token id

  lastUsedAt   DateTime?
  createdAt    DateTime  @default(now())

  @@index([userId])
  @@index([sessionId])
  @@index([expiresAt])
  @@index([revoked])
  @@map("refresh_tokens")
}
```

| Rule | Detail |
|------|--------|
| Store | **Hash only** — `SHA-256(REFRESH_PEPPER + rawToken)` |
| Unique | `tokenHash` globally unique |
| Rotation | Transaction: revoke old (`revoked=true`, `rotatedAt`, `rotatedToId`), insert new |
| Active query | `revoked=false AND expiresAt > now()` |
| Logout-all | `updateMany` set `revoked=true` for `userId` |

**Optional self-relation:**

```prisma
rotatedTo   RefreshToken?  @relation("RefreshRotation", fields: [rotatedToId], references: [id])
rotatedFrom RefreshToken[] @relation("RefreshRotation")
```

## 6. Data flows (post-migration)

### 6.1 Mobile OTP verify

```
MobileOtpChallenge (Prisma or Redis OtpStore)
  → verify OK
  → UserDevice.upsert (if deviceKey)
  → UserSession.create (channel=MOBILE, expiresAt)
  → RefreshToken.create (hash, sessionId)
  → sign access JWT
  → AuthAuditEvent: OTP_VERIFY_SUCCESS
  → response: accessToken + optional refreshToken (plaintext once)
```

### 6.2 Refresh

```
POST /api/auth/token/refresh { refreshToken }
  → hash lookup
  → validate session ACTIVE, token not revoked/expired
  → transaction rotate
  → new access JWT
  → AuthAuditEvent: REFRESH_SUCCESS
```

### 6.3 Logout / logout-all

```
POST /api/auth/logout (Bearer userId)
  → revoke session(s)
  → revoke refresh token(s)
  → Redis session del (if enabled)
  → AuthAuditEvent: LOGOUT, SESSION_REVOKED
```

## 9. Security & compliance

| Topic | Rule |
|-------|------|
| Raw refresh | Never persist; never audit; never log |
| Pepper | `MOBILE_REFRESH_SECRET` or dedicated `REFRESH_TOKEN_PEPPER` |
| Cascade | `onDelete: Cascade` from User — GDPR delete path inherits |
| Metadata | Audit may include `sessionId`, `deviceId` — not token hash |

## 10. Comparison: PHASE1_DB_MAP vs P1-06 decision

| PHASE1_DB_MAP draft | P1-06 decision | Reason |
|---------------------|----------------|--------|
| `sessionHash` on UserSession | **Dropped** | Use `UserSession.id` + optional JWT `sid` claim |
| `context String` on RefreshToken | **`AuthChannel` enum** | Align with panel/mobile channels |
| `revokedAt` only on RefreshToken | **`revoked` boolean + timestamps** | Matches archived rotate queries |
| Separate 3 migrations | **Single migration** | Atomic deploy |
